================================================================
   CONFUSING 'this' KEYWORD — WHEN IT WORKS & WHEN IT DOESN'T
================================================================

This file covers every tricky, confusing, and surprising situation
with 'this' in JavaScript. Every case is explained with examples.


================================================================
  CONFUSING CASE 1 — Method Extracted from Object (BREAKS)
================================================================

SITUATION: You copy a method from an object into a variable and call
it. Now 'this' is lost — it no longer points to the original object.

WHY IT HAPPENS: 'this' depends on HOW the function is called, not
WHERE it was defined. When you extract it, it's called as a plain
function, not as a method.

EXAMPLE:
--------
const user = {
    name: "Ahmad",
    greet: function() {
        console.log("Hello, " + this.name);
    }
};

user.greet();       // ✅ "Hello, Ahmad" — works, this = user
const fn = user.greet;
fn();               // ❌ "Hello, undefined" — this = window (or crashes in strict mode)

FIX:
----
const fn = user.greet.bind(user);
fn(); // ✅ "Hello, Ahmad"


================================================================
  CONFUSING CASE 2 — Callback Functions Lose 'this' (BREAKS)
================================================================

SITUATION: You pass a method as a callback to another function.
The 'this' context is lost inside the callback.

EXAMPLE:
--------
const timer = {
    name: "Clock",
    start: function() {
        setTimeout(function() {
            console.log(this.name); // ❌ undefined — 'this' = window in callback
        }, 500);
    }
};
timer.start();

FIX 1 — Arrow function:
-----------------------
const timer = {
    name: "Clock",
    start: function() {
        setTimeout(() => {
            console.log(this.name); // ✅ "Clock" — arrow inherits 'this'
        }, 500);
    }
};

FIX 2 — bind():
---------------
const timer = {
    name: "Clock",
    start: function() {
        setTimeout(function() {
            console.log(this.name); // ✅ "Clock"
        }.bind(this), 500);
    }
};

FIX 3 — Save 'this' in a variable (old-school):
-------------------------------------------------
const timer = {
    name: "Clock",
    start: function() {
        const self = this; // save reference
        setTimeout(function() {
            console.log(self.name); // ✅ "Clock"
        }, 500);
    }
};


================================================================
  CONFUSING CASE 3 — Arrow Function as Object Method (BREAKS)
================================================================

SITUATION: You write an arrow function directly as an object's
method. It doesn't work as expected because arrow functions don't
have their own 'this' — they look at the outer scope, which is
the global scope (window), not the object.

EXAMPLE:
--------
const person = {
    name: "Fatima",
    greet: () => {                        // ❌ Arrow as method — WRONG
        console.log("Hi, " + this.name); // 'this' = window, not person
    }
};
person.greet(); // "Hi, undefined"

FIX — Use regular function for methods:
----------------------------------------
const person = {
    name: "Fatima",
    greet: function() {                   // ✅ Regular function
        console.log("Hi, " + this.name); // 'this' = person
    }
};
person.greet(); // "Hi, Fatima"


================================================================
  CONFUSING CASE 4 — Nested Regular Functions Inside Methods (BREAKS)
================================================================

SITUATION: Inside a method, you create an inner regular function.
That inner function has its OWN 'this', which defaults to window.

EXAMPLE:
--------
const obj = {
    name: "Outer",
    outer: function() {
        console.log(this.name); // ✅ "Outer" — works fine

        function inner() {
            console.log(this.name); // ❌ undefined — inner has its own 'this'
        }
        inner();
    }
};
obj.outer();

FIX — Use arrow for the inner function:
----------------------------------------
const obj = {
    name: "Outer",
    outer: function() {
        const inner = () => {
            console.log(this.name); // ✅ "Outer" — arrow inherits outer 'this'
        };
        inner();
    }
};
obj.outer();


================================================================
  CONFUSING CASE 5 — Constructor Without 'new' (BREAKS SILENTLY)
================================================================

SITUATION: You call a constructor function WITHOUT using 'new'.
In non-strict mode, 'this' becomes the global object, polluting it.
In strict mode, 'this' is undefined and you get an error.

EXAMPLE (non-strict mode — SILENT BUG):
-----------------------------------------
function Car(brand) {
    this.brand = brand; // 'this' = window — adds 'brand' to window object!
}
const c = Car("Toyota"); // forgot 'new'
console.log(c);           // undefined — Car() returns nothing
console.log(window.brand); // "Toyota" — polluted the global scope!

EXAMPLE (strict mode — CRASH):
--------------------------------
"use strict";
function Car(brand) {
    this.brand = brand; // this = undefined — TypeError!
}
Car("Toyota"); // ❌ TypeError: Cannot set properties of undefined

FIX — Always use 'new' with constructor functions:
---------------------------------------------------
const c = new Car("Toyota"); // ✅
console.log(c.brand); // "Toyota"

BONUS FIX — Self-defending constructor:
-----------------------------------------
function Car(brand) {
    if (!(this instanceof Car)) {
        return new Car(brand); // auto-wraps if called without new
    }
    this.brand = brand;
}
Car("Honda");     // still works safely
new Car("BMW");   // also works


================================================================
  CONFUSING CASE 6 — bind() Cannot Be Re-Bound (PERMANENT)
================================================================

SITUATION: Once you bind 'this' with bind(), no other call(), apply(),
or another bind() can change it. The first bind() wins permanently.

EXAMPLE:
--------
function show() {
    console.log(this.val);
}
const a = { val: 1 };
const b = { val: 2 };
const c = { val: 3 };

const boundToA = show.bind(a);
boundToA();                   // ✅ 1
boundToA.call(b);             // ❌ still 1 — bind() cannot be overridden by call()
boundToA.apply(c);            // ❌ still 1
const boundToB = boundToA.bind(b);
boundToB();                   // ❌ still 1 — second bind() has no effect


================================================================
  CONFUSING CASE 7 — 'this' in Array Methods (TRICKY)
================================================================

SITUATION: When passing a regular function to array methods like
.forEach(), .map(), .filter(), 'this' inside that function is
either window (non-strict) or undefined (strict), NOT the outer object.

EXAMPLE:
--------
const numbers = {
    data: [1, 2, 3],
    multiplier: 10,
    compute: function() {
        return this.data.map(function(n) {
            return n * this.multiplier; // ❌ 'this' = window — this.multiplier = undefined
        });
    }
};
console.log(numbers.compute()); // [NaN, NaN, NaN]

FIX 1 — Arrow function in map:
-------------------------------
compute: function() {
    return this.data.map(n => n * this.multiplier); // ✅ arrow inherits 'this'
}
console.log(numbers.compute()); // [10, 20, 30]

FIX 2 — Pass 'this' as second argument (thisArg):
--------------------------------------------------
compute: function() {
    return this.data.map(function(n) {
        return n * this.multiplier; // ✅ 'this' is set by the second argument
    }, this); // ← pass 'this' as the context
}
// Note: .forEach(), .map(), .filter(), .find(), .some(), .every()
// all accept a thisArg as their SECOND argument.


================================================================
  CONFUSING CASE 8 — Class Methods Passed as Callbacks (BREAKS)
================================================================

SITUATION: In a class, if you pass a method as a callback without
binding it, 'this' is lost.

EXAMPLE:
--------
class Button {
    constructor(label) {
        this.label = label;
    }
    click() {
        console.log("Clicked: " + this.label);
    }
}
const btn = new Button("Submit");
document.addEventListener("click", btn.click);  // ❌ 'this' = window/undefined

FIX 1 — bind in constructor:
------------------------------
constructor(label) {
    this.label = label;
    this.click = this.click.bind(this); // ✅ permanently bound
}

FIX 2 — Arrow function as class field (modern approach):
----------------------------------------------------------
class Button {
    constructor(label) {
        this.label = label;
    }
    click = () => {              // ✅ Arrow field — always bound to instance
        console.log("Clicked: " + this.label);
    }
}
const btn = new Button("Submit");
document.addEventListener("click", btn.click); // ✅ works!


================================================================
  CONFUSING CASE 9 — Chained Method with Detached Call (BREAKS)
================================================================

SITUATION: When you use method chaining through intermediary
objects, 'this' reflects the LAST object before the dot.

EXAMPLE:
--------
const a = {
    val: 1,
    b: {
        val: 2,
        getVal: function() { return this.val; }
    }
};
console.log(a.b.getVal()); // 2 — 'this' = a.b (object immediately before the dot)
// 'this' is NOT 'a', it is 'a.b'


================================================================
  CONFUSING CASE 10 — 'this' in Immediately Invoked Functions (IIFE)
================================================================

SITUATION: An IIFE (immediately invoked function expression) is
called without an object, so 'this' defaults to window or undefined.

EXAMPLE:
--------
const obj = {
    name: "Test",
    run: function() {
        (function() {
            console.log(this.name); // ❌ 'this' = window — IIFE is not a method
        })();
    }
};
obj.run(); // undefined

FIX — Use arrow IIFE:
----------------------
run: function() {
    (() => {
        console.log(this.name); // ✅ "Test" — arrow captures outer 'this'
    })();
}


================================================================
  CONFUSING CASE 11 — Getter/Setter and 'this'
================================================================

SITUATION: 'this' inside getters and setters refers to the object
the property belongs to — this one WORKS as expected.

EXAMPLE:
--------
const circle = {
    radius: 5,
    get area() {
        return Math.PI * this.radius ** 2; // ✅ 'this' = circle
    },
    set diameter(d) {
        this.radius = d / 2; // ✅ 'this' = circle
    }
};
console.log(circle.area);    // 78.54
circle.diameter = 20;
console.log(circle.radius);  // 10


================================================================
  WHEN 'this' WORKS vs. DOESN'T — SUMMARY TABLE
================================================================

SCENARIO                                    | WORKS?  | 'this' IS...
--------------------------------------------|---------|------------------
object.method()                             | ✅ YES  | the object
new Constructor()                           | ✅ YES  | new object
call(obj) / apply(obj) / bind(obj)          | ✅ YES  | obj
Arrow function inside method                | ✅ YES  | inherited from method
Class method called on instance             | ✅ YES  | instance
Getter / Setter in object                   | ✅ YES  | the object
Array methods with thisArg (2nd arg)        | ✅ YES  | thisArg
Event handler (regular function)            | ✅ YES  | the DOM element
--------------------------------------------|---------|------------------
Arrow function as object method             | ❌ NO   | window / global
Constructor called WITHOUT new              | ❌ NO   | window (silent bug!)
Method extracted into variable, then called | ❌ NO   | window / undefined
Regular function as callback                | ❌ NO   | window / undefined
Nested regular function inside method       | ❌ NO   | window / undefined
bind() called again after first bind()      | ❌ NO   | first bind wins
Event handler as arrow function             | ❌ NO   | outer scope (window)
Regular function in .map()/.forEach()       | ❌ NO   | window / undefined
IIFE inside a method                        | ❌ NO   | window / undefined
